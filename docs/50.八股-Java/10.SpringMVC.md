### 1.说一说SpringMVC的执行流程？

:::note

SpringMVC 的执行流程大概是这样的：

首先，用户发起一个请求，会先被 **DispatcherServlet** 拦截到，它其实就是 SpringMVC 的前端控制器。

然后，DispatcherServlet 会根据请求的 URL，去找对应的 **HandlerMapping**，确定这个请求应该交给哪个 Controller 的方法来处理。

找到 Handler 之后，会通过 **HandlerAdapter** 来真正调用 **Controller** 的方法。Controller 方法执行完以后，会返回一个 **ModelAndView**，里面包含了数据模型和要跳转的视图名。

接下来，DispatcherServlet 会把这个 ModelAndView 交给 **ViewResolver**，去解析成具体的视图，比如 JSP、Thymeleaf 之类的。最后把数据渲染到视图里，再返回给用户。

总结一下，就是：SpringMVC 把请求统一交给前端控制器 DispatcherServlet，然后通过映射找到合适的 Controller 方法，执行完以后再通过视图解析器去渲染，最后把结果返回给前端。

:::
