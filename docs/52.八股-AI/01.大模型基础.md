我们通过聊天框发送一条消息给AI模型，，然后AI模型生成一个回复。我们发的消息就叫User Prompt 也就是用户提示词，一般就是我们提出的问题或者想说的话。

但是现实生活中，当我们和不同人聊天时，即便是完全相同的话，对方也会根据自己的经验给出不同的答案。

比如，我说我肚子疼

我妈可能会问我要不要去医院看看

我爸可能会让我去厕所

我女朋友可能会说一句：滚一边去，老娘也疼

但是AI并没有这样的人设，所以它只能给出一个通用的四平八稳的回答：请就医。

显得非常无趣，因此我们希望给AI也加上人设，最直接的方法就是把人设信息和用户要说的话，打包成一条User Prompt发过去，比如你扮演我的女朋友，我说我肚子疼，然后AI就可能回复，滚一边去，老娘也疼。

但问题是你扮演我温柔的女朋友，这句话并不是我们真正想说的内容，于是人们干脆把人设信息单独拎出来，放到另外一个Prompt里面，这就是System Prompt 系统提示词。

System Prompt 主要用来描述AI的角色、性格、背景信息、语气等等。总之只要不是用户直接说出来的内容，都可以放进System Prompt 里面。每次用户发送User Prompt 的时候，系统会自动把System Prompt也一起发送给AI模型。

这样整个对话就显得更加自然了，

AI只是一个聊天机器人，你问一个问题，它最多给你答案或者告诉你怎么做，但实际动手的还是你自己，那么能不能让AI自己去完成任务呢？

比如我们想让AI自动帮我们管理电脑的文件，我们可以先写好一些文件的管理函数，比如说list_file用来列目录、read_file用来读文件，然后把这些函数以及它们的功能描述、使用方法注册到AI agent中，AI Agent会根据这些信息生成一个System Prompt ，告诉AI模型用户给了你哪些工具，它们都是干什么的，以及AI如果想要使用它们，应该返回什么样的格式，最后把这个System Prompt 连同用户的请求，比如说帮我找一找原神的安装目录一起发给AI，如果AI模型足够聪明，就会按照要求的格式，返回一个调用某个函数的信息，AI agent进行解析之后，就可以调用对应的函数了，然后再把结果丢回给AI，AI再根据函数调用的结果决定下一步应该做什么操作。

提供给AI调用的函数或者服务，就叫做Agent Tools

Function Calling：统一格式、规范描述，将每个Tool都用一个Json对象来定义。

然后这些对象也从System Prompt中被剥离了出来，单独放到了一个字段里面，

Function Calling也规定了AI使用工具时应该返回的格式。

AI Agent如何与Tools进行通信的，MCP是一个通信协议，专门用来规范Agent和Tool服务之间是怎么交互的，运行Tool的服务叫做MCP Server，调用它的Agent叫做MCP Client。

MCP规定了MCP Server如何和MCP Client通信，以及MCP Server要提供哪些接口
