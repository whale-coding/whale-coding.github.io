## docker搭建后端常见开发环境

### 0.常见docker网络

```bash
# 创建docker网络，否则容器之间没法用名字互通
docker network create dev-net

# 查看docker网络
docker network ls
```

### 1.MySQL

```bash
# 1.创建目录，用于容器数据的持久化
mkdir -p ~/docker/mysql/{conf,data}

# 2.拉取镜像
docker pull mysql:8.0.29

# 3.启动容器
docker run -d \
--name mysql \
--network dev-net \
-p 3307:3306 \
-e MYSQL_ROOT_PASSWORD=root \
-v ~/docker/mysql/data:/var/lib/mysql \
-v ~/docker/mysql/conf:/etc/mysql/conf.d \
mysql:8.0.29
```

**启动命令含义：**

- `-d`：后台运行
- `--name mysql`：容器命名为 `mysql`
- `--network dev-net`：连接到自定义网络 `dev-net`
- `-p 3307:3306`：**宿主机 3307 端口映射到容器 3306 端口**
- `-e MYSQL_ROOT_PASSWORD=root`：设置 root 密码为 `root`
- `-v ~/docker/mysql/data:/var/lib/mysql`：**数据持久化** - 将 MySQL 数据文件映射到宿主机目录
- `-v ~/docker/mysql/conf:/etc/mysql/conf.d`：**配置持久化** - 将配置文件映射到宿主机目录
- `mysql:8.0.29`：使用 MySQL **8.0.29** 版本镜像

### 2.Redis

```bash
# 1.创建目录，用于容器数据的持久化
mkdir -p ~/docker/redis/data

# 2.拉取镜像
docker pull redis:7.2

# 3.启动容器
docker run -d \
--name redis \
--network dev-net \
-p 6380:6379 \
-v ~/docker/redis/data:/data \
redis:7.2 \
redis-server --appendonly yes --requirepass 123456
```

**启动命令含义：**

- `-d`：后台运行
- `--name redis`：容器命名为 `redis`
- `--network dev-net`：连接到自定义网络 `dev-net`
- `-p 6380:6379`：**宿主机 6380 端口映射到容器 6379 端口**
- `~/docker/redis/data:/data`：**数据持久化** - 将 Redis 数据文件映射到宿主机目录
- `redis:7.2`：使用 Redis **7.2** 版本镜像
- `--appendonly yes`：开启 AOF 持久化
- `--requirepass 123456`：设置密码

### 3.Nacos

```bash
# 1.创建目录，用于容器数据的持久化
mkdir -p ~/docker/nacos/{logs,data}

# 2.拉取镜像
docker pull nacos/nacos-server:v2.1.1-slim  # -slim表示专门的 ARM 镜像

# 3.启动容器
docker run -d \
--name nacos \
--network dev-net \
-p 8848:8848 \
-p 9848:9848 \
-p 9849:9849 \
-e MODE=standalone \
-e TZ=Asia/Shanghai \
-v ~/docker/nacos/logs:/home/nacos/logs \
-v ~/docker/nacos/data:/home/nacos/data \
nacos/nacos-server:v2.1.1-slim
```

**启动命令含义：**

- `-d`：后台运行
- `--name nacos`：容器命名为 `nacos`
- `--network dev-net`：连接到自定义网络 `dev-net`
- `-p 8848:8848`：**宿主机 8848 端口映射到容器 8848 端口**
- `-e MODE=standalone`：以**单机模式**运行
- `-e TZ=Asia/Shanghai`：设置容器时区为上海
- `-v ~/docker/nacos/logs:/home/nacos/logs`：日志持久化， 将容器日志映射到宿主机
- `-v ~/docker/nacos/data:/home/nacos/data`：数据持久化， 将容器数据映射到宿主机
- `nacos/nacos-server:v2.1.1-slim`：使用Nacos **2.1.1** 版本ARM镜像

web界面访问地址:http://localhost:8848/nacos

- **用户名**：`nacos`
- **密码**：`nacos`

### 4.Kafka

```bash
# 1.创建目录
mkdir -p ~/docker/kafka/data ~/docker/kafka/config

# 2.拉取镜像
docker pull apache/kafka:4.0.0   # 目前最先版本，不需要ZooKeeper的依赖

# 3.启动容器
docker run -d \
--name kafka \
--network dev-net \
-p 9092:9092 \
-p 9093:9093 \
-e CLUSTER_ID=5L6g3nShT-eMCtK--X86sw \
-e KAFKA_PROCESS_ROLES=broker,controller \
-e KAFKA_NODE_ID=1 \
-e KAFKA_CONTROLLER_QUORUM_VOTERS=1@kafka:9093 \
-e KAFKA_LISTENERS=PLAINTEXT://:9092,CONTROLLER://:9093 \
-e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092 \
-e KAFKA_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT \
-e KAFKA_CONTROLLER_LISTENER_NAMES=CONTROLLER \
-e KAFKA_INTER_BROKER_LISTENER_NAME=PLAINTEXT \
-e KAFKA_AUTO_CREATE_TOPICS_ENABLE=true \
-e KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR=1 \
-e KAFKA_TRANSACTION_STATE_LOG_REPLICATION_FACTOR=1 \
-e KAFKA_TRANSACTION_STATE_LOG_MIN_ISR=1 \
-v ~/docker/kafka/data:/var/lib/kafka/data \
apache/kafka:4.0.0
```

**启动命令含义：**

- `-d`：后台运行
- `--name kafka`：容器命名为 `kafka`
- `--network dev-net`：连接到自定义网络 `dev-net`
- `-p 9092:9092`：宿主机 9092 端口映射到容器 9092 端口（Kafka 客户端通信端口）
- `-p 9093:9093`：宿主机 9093 端口映射到容器 9093 端口（KRaft 控制器通信端口）
- `-e CLUSTER_ID=5L6g3nShT-eMCtK--X86sw`：设置集群唯一标识 ID（KRaft 模式必需）
- `-e KAFKA_PROCESS_ROLES=broker,controller`：当前节点同时充当 Broker 和 Controller（KRaft 模式）
- `-e KAFKA_NODE_ID=1`：设置节点 ID 为 1
- `-e KAFKA_CONTROLLER_QUORUM_VOTERS=1@kafka:9093`：控制器投票者列表，`1@kafka:9093` 表示节点 1 在 `kafka:9093` 上参与选举
- `-e KAFKA_LISTENERS=PLAINTEXT://:9092,CONTROLLER://:9093`：定义监听地址：PLAINTEXT 协议在 9092，CONTROLLER 协议在 9093
- `-e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092`：对外宣告的监听地址，客户端通过 `localhost:9092` 连接
- `-e KAFKA_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT`：为监听器指定安全协议，都使用 PLAINTEXT（明文传输）
- `-e KAFKA_CONTROLLER_LISTENER_NAMES=CONTROLLER`：指定控制器使用的监听器名称为 `CONTROLLER`
- `-e KAFKA_INTER_BROKER_LISTENER_NAME=PLAINTEXT`：Broker 之间通信使用 `PLAINTEXT` 监听器
- `-e KAFKA_AUTO_CREATE_TOPICS_ENABLE=true`：允许自动创建 Topic（生产环境建议关闭）
- `-e KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR=1`：`__consumer_offsets` 主题副本数设为 1（单节点必须设为 1）
- `-e KAFKA_TRANSACTION_STATE_LOG_REPLICATION_FACTOR=1`：事务日志副本数设为 1（单节点必须设为 1）
- `-e KAFKA_TRANSACTION_STATE_LOG_MIN_ISR=1`：事务日志最小同步副本数设为 1（单节点必须设为 1）
- `-v ~/docker/kafka/data:/var/lib/kafka/data`：数据持久化，将 Kafka 数据文件映射到宿主机目录
- `apache/kafka:4.0.0`：使用 Apache Kafka 官方镜像 4.0.0 版本

### 5.Elasticsearch

```bash
# 1.建目录，后面把 IK 插件放这里，升级/重启容器都不会丢
mkdir -p ~/docker/elasticsearch/{data,plugins,logs,conf}

# 2.拉取elasticsearch镜像
docker pull elasticsearch:8.14.0

# 3.下载对应版本的ik中文分词器
https://release.infinilabs.com/analysis-ik/stable/
# 下载之后放到~/docker/elasticsearch/plugins 目录下

# 4.制作配置文件，关闭 8.x 默认安全
cat <<EOF >~/docker/elasticsearch/conf/elasticsearch.yml
cluster.name: docker-cluster
network.host: 0.0.0.0
http.port: 9200
discovery.type: single-node
xpack.security.enabled: false        # 关掉账号密码+TLS
xpack.security.transport.ssl.enabled: false
EOF

# 5.启动容器
docker run -d \
--name elasticsearch \
-p 9200:9200 \
-p 9300:9300 \
--network dev-net \
-e "discovery.type=single-node" \
-e ES_JAVA_OPTS="-Xms1024m -Xmx1024m" \
-v ~/docker/elasticsearch/conf/elasticsearch.yml:/usr/share/elasticsearch/config/elasticsearch.yml \
-v ~/docker/elasticsearch/data:/usr/share/elasticsearch/data \
-v ~/docker/elasticsearch/plugins:/usr/share/elasticsearch/plugins \
elasticsearch:8.14.0

# 6.测试
# 6.1 浏览器访问如下地址，返回es的版本等信息，即安装成功
http://127.0.0.1:9200 

# 6.2 浏览器访问如下地址，返回行里能看到 analysis-ik 即安装成功分词器成功
http://127.0.0.1:9200/_cat/plugins?v
```

快速测试中文分词：

```bash
curl -X POST "localhost:9200/_analyze" -H 'Content-Type: application/json' -d'
{
  "analyzer": "ik_max_word",
  "text": "中华人民共和国国歌"
}'

```

### 6.Kibana

```bash
# 1.建目录
mkdir -p ~/docker/kibana/{data,conf}

# 2.拉取kibana镜像，注意需要跟es版本保持一致！
docker pull kibana:8.14.0

# 3.创建配置文件
cat <<EOF> ~/docker/kibana/conf/kibana.yml
server.host: "0.0.0.0"  # 不配报错
server.shutdownTimeout: "5s"
elasticsearch.hosts: [ "http://elasticsearch:9200" ]
elasticsearch.username: "kibana_system"  # 不能用 elastic 
elasticsearch.password: "111111"
i18n.locale: "zh-CN"
EOF

# 4.启动容器
docker run -d \
--name kibana \
--network dev-net \
-v ~/docker/kibana/conf/kibana.yml:/usr/share/kibana/config/kibana.yml \
-p 5601:5601
kibana:8.14.0

# 5.测试
# 浏览器访问如下地址，
http://127.0.0.1:5601
```
