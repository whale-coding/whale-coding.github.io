>如果简历向量数据库使用的是pgvect参考回答：

项目选择使用PostgreSQL+pgvect 的主要原因包括：

1. 我们的项目属于小型项目，数据规模不大，向量数据规模小于百万级别，使用专业向量数据库属于大材小用；
2. 使用专业的向量数据库运维成本比较高，使用PostgreSQL+pgvect 的学习、运维成本比较低；
3. PostgreSQL+pgvect 方案向量数据和业务数据可以在同一个事务中进行管理，天然支持事务；
4. 可以结合 SQL 过滤条件进行结构化+向量联合查询。

如果未来向量数据规模突破千万级，可以再考虑迁移到专业的向量数据库，比如Milvus。

>如果简历向量数据库使用的是Elasticsearch参考回答：

项目选择使用Elasticsearch的主要原因包括：

1. 我的技术栈主要为 Java，对ES 比较熟悉、学习成本相对较低，对于其他项目也会用到 ES，因此使用 ES 作为向量数据库可以复用现有的基础设施；
2. Elasticsearch的混合检索能力比较强，既支持既支持传统的全文检索，也支持向量检索，可无缝结合 BM25 关键词检索和向量检索。
3. Elasticsearch 还支持过滤和聚合，强大的 DSL 语句支持各种复杂的查询。
4. Elasticsearch 原生分布式架构，未来可以很方便的进行水平扩展。

如果未来向量数据规模突破千万级，可以再考虑迁移到专业的向量数据库，比如Milvus。
