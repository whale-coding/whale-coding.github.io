### 1.cookie和session的区别？

参考资料：

https://www.cnblogs.com/ityouknow/p/10856177.html

https://blog.csdn.net/weixin_45393094/article/details/104747360

:::note

- Cookie 存储在客户端（浏览器），而Session 存储在服务器端
- Session比Cookie更安全
- Cookie 只能保存 ASCII，Session 可以存任意数据类型
- Cookie 可设置为长时间保持，Session 一般失效时间较短，客户端关闭会失效
- 单个 Cookie 保存的数据不能超过 4K，Session 可存储数据的大小与服务器的内存有关

:::

### 2.过滤器和拦截器的区别？

参考资料：

https://www.cnblogs.com/vipstone/p/16796867.html

https://www.cnblogs.com/summerday152/p/13658788.html

:::note

- 过滤器来自于 Servlet，而拦截器来自于 Spring 框架；
- 过滤器是基于函数回调实现，拦截器是基于Java的反射机制实现；
- 过滤器对请求进入Servlet前后进行处理，拦截器对请求进入Controller前后进行处理，所以过滤器会先执行，然后才会执行拦截器，最后才会进入真正的要调用的方法；
- 过滤器依赖 Servlet 容器，只能用在 Web 项目中，拦截器即可以用于web项目也可以用于非web项目；
- 过滤器通常用来实现通用功能过滤，如敏感词过滤，拦截器主要用来实现项目中的业务判断，如登录判断、权限判断；

:::
