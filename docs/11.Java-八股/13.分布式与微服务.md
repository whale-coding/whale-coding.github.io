## 分布式理论

### ⭐️1.什么是CAP定理？

:::note 参考答案

CAP定理：在分布式系统中不可能同时满足一致性（Consistency）、可用性（Availability）和分区容错性（Partition tolerance）这三个特性，最多只能同时满足其中两个。

- **一致性**：指的是所有节点在同一时间看到的数据都是一致的。
- **可用性**：指的是系统在正常响应时间内返回合理的响应，即使部分节点出现故障，系统仍然可以继续提供服务。
- **分区容错性**：指的是当网络分区故障发生时，系统仍然能够继续运行。网络分区是分布式系统中不可避免的问题，比如机房之间的网络中断，这时候不同机房的节点就无法正常通信了。

对于分布式系统来说，P 是必须要有的，因为网络既然会存在分区问题（网络延迟、丢包、中断等），分区容错性也就成为了必然的选择。因此，对于分布式系统来说，要么是CP，要么是AP。

:::

>扩展知识点1：什么是网络分区？

分布式系统中，多个节点之间的网络本来是连通的，但是因为某些故障（比如部分节点网络出了问题）某些节点之间不连通了，整个网络就分成了几块区域，这就叫**网络分区**。

### 2.为什么 CAP 不可兼得？

:::note 参考答案

对于分布式系统来说，分区是必然存在的。网络分区指的是在分布式系统中，多个节点之间的网络本来是连通的，但是因为某些故障（比如部分节点网络出了问题）某些节点之间不连通了，整个网络就分成了几块区域。

因此，分区容错性（P）必须要满足，因为如果要牺牲分区容错性，就得把服务和资源放到一个机器，或者一个“同生共死”的集群，那就违背了分布式的初衷。

在分区容错的前提下，`一致性`和`可用性`是矛盾的，只能满足其一。

:::

### 3.什么是 BASE 理论？

:::note 参考答案

BASE（Basically Available、Soft state、Eventual consistency）理论是对CAP定理中 AP 方案的延伸，核心思想是即便不能达到强一致性，也可以根据应用特点采用适当的方式来达到最终一致性的效果。

BASE 的主要含义：

- **Basically Available（基本可用）**：系统出现故障时，允许损失部分可用性（如响应时间增加、功能降级）。
- **Soft State（软状态）**：允许系统存在中间状态，该状态不影响整体可用性。即允许系统在多个不同节点的数据副本存在数据延时。
- **Eventually Consistent（最终一致性）**：系统保证在一段时间内达到数据的最终一致。

:::

## 分布式事务

### 1.什么是分布式事务？

:::note 参考答案

分布式事务，本质上就是为了解决在微服务架构下，因业务操作跨越了多个独立的服务和数据库，而无法使用本地事务来保证数据一致性的问题。



分布式事务是指在分布式系统环境下，一次业务操作涉及多个服务、多个数据库节点或多个资源服务器，这些操作需要共同保证数据的一致性，即“要么全部成功，要么全部失败”。它解决了微服务架构下跨服务调用导致本地事务失效的问题。

:::
